# 🛒 E-Commerce Shopping Application – Manual Testing Project

## 📌 Project Overview

This project demonstrates the manual testing process for an E-Commerce Shopping Application.  
The objective of this project is to validate the functionality, reliability, and usability of the application by designing and executing test plans, test cases, and reporting defects.

This project covers major modules such as Homepage, Product Details, Cart Management, Checkout, and User Account Management.

---

## 🎯 Objectives

- To design a comprehensive Test Plan for the shopping application.
- To create detailed Test Cases covering all functional modules.
- To execute test cases and identify defects.
- To apply test design techniques like Boundary Value Analysis (BVA) and Equivalence Partitioning (EP).
- To ensure application quality through Functional, Integration, and Regression Testing.

---

## 🧩 Modules Covered

### 1️⃣ Homepage
- Product display (3-column layout)
- Sorting functionality
- Filtering functionality
- Pagination (10, 25, 50 products per page)

### 2️⃣ Product Details Page
- Product description validation
- Model number and manufacturer fields
- Social media sharing buttons
- Feedback form validation

### 3️⃣ Cart Management
- Add product to cart
- Remove product from cart
- Increase/Decrease quantity
- Cart total calculation validation

### 4️⃣ Checkout & Payment
- Billing address validation
- Shipping address validation
- Coupon/Gift card functionality
- Order placement process

### 5️⃣ User Account Management
- User Signup
- User Login
- Dashboard verification
- Address management
